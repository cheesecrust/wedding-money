# 축의금 관리 앱 요구사항 문서

> 작성일: 2026-03-28  
> 기반: 대화 내용 기반 요구사항 정리

---

## 1. 프로젝트 개요

결혼식 당일 축의금을 빠르게 입력하고, 신랑과 공유할 수 있는 경량 웹앱.  
설치 불필요, 오프라인 동작, 브라우저 하나로 완결.

---

## 2. 최종 결정 사항

| 항목 | 결정 |
|------|------|
| 플랫폼 | 로컬 HTML 파일 (단일 파일 웹앱) |
| 데이터 저장 | LocalStorage (브라우저 자동 저장) |
| 공유 방식 | 내보내기 버튼 → `.html` 파일 → 카톡 전송 |
| 오프라인 지원 | ✅ 완전 지원 |
| 설치 필요 여부 | ❌ 없음 |

### 검토하고 제외한 방안

- **Google Sheets + 앱스크립트** — 서버 통신으로 인한 1~3초 딜레이, 오프라인 불가
- **Excel + VBA** — Mac에서 VBA 불안정, `.xlsm` 파일 오류 발생
- **Supabase 백엔드** — DB 설정 복잡, 인터넷 필수
- **JSON 파일 공유** — 매번 파일 전송이 번거로움

## 3. 기능 요구사항

### 3-1. 입력 기능 (필수)

- [ ] 이름 / 축의금 / 식권 개수 — 3개 입력 필드
- [ ] 각 필드에서 **Enter** 키로 다음 필드 이동
- [ ] 마지막 필드(식권 개수)에서 Enter → 즉시 행 추가
- [ ] 추가 후 입력 필드 자동 초기화 및 이름 필드로 포커스 복귀
- [ ] 추가 완료 시 토스트 메시지 표시

### 3-2. 명단 테이블 (필수)

| 컬럼 | 비고 |
|------|------|
| 번호 | 자동 증가 |
| 이름 | 입력값 |
| 관계 | 드롭다운: 친구 / 직장동료 / 친척 / 선후배 / 기타 |
| 축의금 | 숫자, 원 단위 포맷 |
| 식권 개수 | 숫자 |
| 답례품 | 드롭다운: O / X |
| 전달방법 | 드롭다운: 직접전달 / 계좌이체 / 모바일 |
| 메모 | 텍스트 입력 |
| 삭제 | 행 삭제 버튼 |

- [ ] 짝수/홀수 행 줄무늬 스타일
- [ ] 하단 합계행 (총 금액 + 총 식권)
- [ ] 삭제 시 확인 다이얼로그

### 3-3. 요약 카드 (필수)

- [ ] 총 축의금 합계
- [ ] 총 식권 개수
- [ ] 총 인원
- [ ] 1인 평균 축의금
- [ ] 입력/삭제 즉시 자동 갱신

### 3-4. 검색 및 필터 (필수)

- [ ] 이름 실시간 검색 (입력하는 즉시 필터링)
- [ ] 관계별 필터 버튼: 전체 / 친구 / 직장동료 / 친척 / 기타

### 3-5. 데이터 저장 (필수)

- [ ] LocalStorage 자동 저장 (입력/수정/삭제마다 즉시 저장)
- [ ] 브라우저 재시작 후에도 데이터 유지

### 3-6. 공유용 내보내기 (필수)

- [ ] "📤 공유용 내보내기" 버튼
- [ ] 클릭 시 데이터가 포함된 독립 `.html` 파일 다운로드
- [ ] 내보낸 파일에 포함되는 내용:
  - 요약 카드 (총 축의금 / 식권 / 인원 / 평균)
  - 전체 명단 테이블
  - 관계별 분석 테이블
  - **이름 검색 기능** (JS 포함)
- [ ] 내보낸 파일은 읽기 전용 (입력 기능 없음)
- [ ] 파일명: `축의금_명단_YYYYMMDD.html`

## 4. 비기능 요구사항

| 항목 | 내용 |
|------|------|
| 오프라인 | 인터넷 없이 완전 동작 |
| 플랫폼 | Mac / Windows 브라우저 (Chrome 권장) |
| 설치 | 불필요 — HTML 파일 열기만으로 실행 |
| 응답속도 | 입력 즉시 반응 (서버 통신 없음) |
| 공유 | 단일 `.html` 파일로 카톡/메일 전송 가능 |
